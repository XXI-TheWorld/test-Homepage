## Context

当前 guest ID 是 4 位数字，游客希望能使用更有意义的自定义名称。需要修改 ID 生成和验证逻辑。

## Goals / Non-Goals

**Goals:**
- 支持最多 15 个中/英文字符的自定义 ID
- 检查 ID 唯一性
- 重复 ID 提示"该名称已被占用"

**Non-Goals:**
- 不实现用户注册系统
- 不追溯更新历史文章/评论的作者名称

## Decisions

### 1. ID 格式
使用 prompt 输入自定义名称，限制 1-15 个字符，支持中英文。不再使用随机数字。

### 2. 唯一性检查
在设置新 ID 时，检查 localStorage 中所有文章的作者昵称和评论昵称是否已存在该名称。若存在，提示"该名称已被占用"。

### 3. 验证方式
修改现有 guest ID 存储机制，使用 prompt 获取用户输入，不再有固定格式限制。

## Risks / Trade-offs

- **历史数据**: 旧文章的作者昵称仍是 4 位数字，这是可接受的
- **ID 冲突**: 游客修改 ID 后可能与他人相同，需检查唯一性
