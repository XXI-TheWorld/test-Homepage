## Context

当前文章评论系统要求游客手动输入昵称才能发表评论。游客每次访问都需要重新输入，而且无法跨会话保持身份一致性。

## Goals / Non-Goals

**Goals:**
- 实现游客 ID 自动生成并持久化存储
- 页面加载时自动恢复游客身份
- 简化评论流程，无需手动输入昵称

**Non-Goals:**
- 不实现用户注册/登录系统
- 不提供昵称修改功能（4 位 ID 固定）
- 不支持多设备同步身份

## Decisions

### 1. 存储结构
采用 localStorage 存储 `guest_id` 键值，值为 4 位数字字符串。

### 2. ID 生成规则
随机生成 1000-9999 之间的整数，确保 4 位数字。简单直接，满足需求。

### 3. UI 显示
在评论区顶部显示"当前身份：XXXX"，readonly 显示，不提供输入框。

### 4. 评论提交逻辑
提交评论时，直接使用 localStorage 中的 guest_id 作为 nickname，不再从表单读取。

## Risks / Trade-offs

- **隐私风险**: 4 位 ID 无法追溯真实用户，风险较低
- **ID 冲突**: 1000-9999 范围（9000 个）冲突概率极低
