## Context

当前系统游客写文章需要手动输入昵称，与评论系统已实现的自动 guest ID 不一致。删除操作时需要额外询问昵称确认身份，体验可以更顺畅。

## Goals / Non-Goals

**Goals:**
- 写文章时自动使用 guest ID
- 文章作者可以删除自己的文章
- 游客可以修改自己的 guest ID
- 删除时直接使用已知 guest ID 验证

**Non-Goals:**
- 不实现用户注册/登录系统
- 不支持多设备同步

## Decisions

### 1. Guest ID 修改机制
提供"修改身份"按钮，点击后通过 prompt 输入新 ID（1000-9999），保存到 localStorage。评论和文章的作者信息不会追溯更新，仅新的评论/文章使用新 ID。

### 2. 文章删除验证
利用当前 guest ID 与文章作者的匹配直接验证，无需额外输入。

### 3. 评论删除验证
利用当前 guest ID 与评论作者的匹配直接验证，无需额外输入。

## Risks / Trade-offs

- **修改 ID 后历史不可追溯**: 游客修改 ID 后，之前的文章/评论仍显示旧 ID，这是可接受的设计
- **ID 冲突**: 随机生成的新 ID 可能与他人相同，但概率极低
