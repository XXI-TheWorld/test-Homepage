## Context

现有项目是一个个人主页，包含一个访客评论区（comments.html），游客可以发表简短留言。当前留言直接显示内容，不适合长篇文章阅读。

用户希望升级为博客系统：游客可以写文章（标题+正文），文章列表只显示标题链接，点击标题进入文章详情页阅读全文。

## Goals / Non-Goals

**Goals:**
- 提供文章发布功能（标题 + 正文）
- 将现有评论区重构为文章列表页
- 新增文章详情页展示完整文章内容
- 保持与主页一致的视觉风格
- 复用 localStorage 本地存储

**Non-Goals:**
- 不实现文章编辑/删除功能
- 不实现文章分类/标签
- 不实现评论功能（仅展示文章）
- 不实现后端服务器

## Decisions

### 1. 重构 comments.html 为文章列表页
**决定**: 将现有的 comments.html 评论区页面重构为文章列表页
**原因**: 用户明确要求"评论区只显示标题链接"，原留言展示需要升级；避免增加新页面复杂度
**替代方案**: 考虑新建 `posts.html` 列表页，但增加页面数量

### 2. 新建 article.html 作为文章详情页
**决定**: 创建独立的 `article.html` 页面展示文章全文
**原因**: 文章内容可能较长，需要独立页面提供更好的阅读体验；URL 参数传递文章 ID
**替代方案**: 考虑过模态框展示，但会影响阅读体验

### 3. 使用 URL 参数传递文章 ID
**决定**: 通过 `article.html?id=xxx` URL 参数识别文章
**原因**: 简单直接，无需复杂路由系统；适合静态页面
**替代方案**: 考虑过 localStorage 传递，但刷新后状态丢失

### 4. localStorage 分离存储
**决定**: 文章数据存储在 `visitorArticles` 键，与原 `visitorComments` 分离
**原因**: 保持数据结构清晰；未来可能需要独立评论功能
**替代方案**: 考虑过合并存储，但会增加复杂度

### 5. 复用现有样式和组件
**决定**: 复用 wood-grain 边框、主题切换、返回链接等组件
**原因**: 保持品牌一致性；减少重复代码
**替代方案**: 独立设计，但会破坏视觉统一性

## Risks / Trade-offs

| 风险 | 影响 | Mitigation |
|------|------|-------------|
| localStorage 5MB 限制 | 长篇文章可能占用较多空间 | 限制单篇文章长度（10000字符） |
| 文章数量增多 | 列表加载变慢 | 仅展示标题列表，加载快 |
| URL 参数暴露 | 用户可能手动修改 ID 查看不存在的文章 | 显示"文章不存在"提示 |

## Open Questions

- 是否需要限制文章数量（如最多50篇）？
- 是否需要显示作者昵称？
- 文章时间显示格式是否与列表保持一致？
