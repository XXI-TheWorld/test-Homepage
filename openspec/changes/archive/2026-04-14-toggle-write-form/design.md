## Context

现有文章列表页 (`comments.html`) 在页面加载时直接显示写文章表单，用户打开页面首先看到的是表单而不是文章列表。用户希望先浏览文章列表，有写作欲望时再点击"我来说两句"按钮展开表单。

同时页面宽度只有 520px，用户反馈太窄，希望加宽到约 960px（桌面端）。

## Goals / Non-Goals

**Goals:**
- 添加"我来说两句"按钮，点击展开写作表单
- 表单默认隐藏，页面加载时只显示文章列表
- 表单收起后恢复到列表状态
- 页面容器宽度增加到 960px

**Non-Goals:**
- 不改变文章列表的展示逻辑
- 不改变表单验证逻辑
- 不改变 localStorage 存储逻辑

## Decisions

### 1. 使用 JavaScript 控制表单显示/隐藏
**决定**: 通过添加/移除 CSS 类控制表单可见性
**原因**: 简单直接，无需复杂状态管理；符合渐进增强原则
**替代方案**: 考虑过 `display: none` vs `visibility: hidden`，选择前者更合适

### 2. 按钮固定在页面底部
**决定**: "我来说两句"按钮固定在页面可见区域底部
**原因**: 始终可见，方便用户随时点击；符合移动端设计惯例
**替代方案**: 考虑过浮动按钮，但固定位置更稳定

### 3. 容器宽度调整为 960px
**决定**: 最大宽度从 520px 调整为 960px
**原因**: 用户明确反馈太窄，960px 是常见的博客宽度
**替代方案**: 考虑过 800px，但 960px 留出更多空间

### 4. 使用 CSS transition 实现平滑动画
**决定**: 表单展开/收起添加平滑过渡动画
**原因**: 提升用户体验，避免突兀的布局跳动
**替代方案**: 无动画，但用户体验不如有动画

## Risks / Trade-offs

| 风险 | 影响 | Mitigation |
|------|------|-------------|
| 表单隐藏时用户不知道可以写作 | 用户可能不会发现写作功能 | 按钮始终可见，有引导文案 |
